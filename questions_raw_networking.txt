EASY LEVEL (Questions 1-100)
Basic Networking Concepts (1-10)
1. What does LAN stand for?
Local Area Network - a network covering a small geographic area like
a home, o ce, or building, typically owned and managed by a single
organization.
2. What does WAN stand for?
Wide Area Network - a network spanning large geographic areas,
connecting multiple LANs across cities, countries, or continents.
3. What is the Internet?
A global network of interconnected networks using standardized
protocols (TCP/IP) to share information and resources worldwide.
4. What does IP stand for?
Internet Protocol - the network layer protocol responsible for
addressing and routing packets across networks.
5. What is an IP address?
A unique numerical identi er (e.g., 192.168.1.1) assigned to devices
on a network for identi cation and communication.
6. What does TCP stand for?
Transmission Control Protocol - a reliable, connection-oriented
transport layer protocol ensuring data arrives complete and in order.
7. What does UDP stand for?
User Datagram Protocol - a connectionless, unreliable transport
protocol used when speed is more important than reliability.
8. What is a protocol?
A set of rules and standards governing how data is transmitted and
received over a network.
9. What is bandwidth?
The maximum amount of data that can be transmitted over a
network connection in a given time, typically measured in bits per
second (bps).
10. What is latency?
The time delay between sending data and receiving it, measured in
milliseconds (ms).
Network Devices (11-20)
11. What is a router?
A network device that forwards data packets between di erent
networks, determining the best path using IP addresses.
12. What is a switch?
A network device that connects devices within a LAN, forwarding
data using MAC addresses at the data link layer.
13. What is a hub?
A basic network device that broadcasts data to all connected devices
(obsolete, replaced by switches).
14. What is a modem?
A device that modulates and demodulates signals, converting digital
data to analog for transmission over phone/cable lines and vice
versa.
15. What is a network interface card (NIC)?
Hardware that enables a computer to connect to a network, either
wired (Ethernet) or wireless (Wi-Fi).
16. What is an access point?
A device that creates a wireless network by connecting to a wired
network, allowing Wi-Fi-enabled devices to connect.
17. What is a gateway?
A device connecting networks using di erent protocols, translating
between them to enable communication.
18. What is a rewall?
A security device or software that monitors and controls network
tra c based on security rules, protecting against unauthorized
access.
19. What is a repeater?
A device that ampli es or regenerates signals to extend network
transmission distance.
20. What is a bridge?
A device connecting two network segments, ltering tra c based on
MAC addresses to reduce congestion.
IP Addressing (21-30)
21. What is IPv4?
Internet Protocol version 4 - uses 32-bit addresses (e.g., 192.168.1.1),
providing approximately 4.3 billion unique addresses.
22. What is IPv6?
Internet Protocol version 6 - uses 128-bit addresses (e.g.,
2001:0db8::1), providing vastly more addresses than IPv4.
23. What is a subnet mask?
A 32-bit number dividing an IP address into network and host
portions (e.g., 255.255.255.0).
24. What is a private IP address?
IP addresses reserved for use within private networks, not routable
on the public Internet (e.g., 192.168.x.x, 10.x.x.x).
25. What is a public IP address?
A globally unique IP address assigned to devices directly accessible
from the Internet.
26. What is DHCP?
Dynamic Host Con guration Protocol - automatically assigns IP
addresses and network con guration to devices on a network.
27. What is a static IP address?
A manually assigned, permanent IP address that doesn't change.
28. What is a dynamic IP address?
An IP address automatically assigned by DHCP that may change over
time.
29. What is NAT?
Network Address Translation - allows multiple devices on a private
network to share a single public IP address.
30. What is a loopback address?
IP address 127.0.0.1 (localhost) used to test network software without
sending packets over the network.
DNS and Domain Names (31-40)
31. What does DNS stand for?
Domain Name System - translates human-readable domain names
(e.g., google.com) into IP addresses.
32. What is a domain name?
A human-readable address identifying a website or resource on the
Internet (e.g., example.com).
33. What is a URL?
Uniform Resource Locator - the complete web address including
protocol, domain, and path (e.g., https://www.example.com/page).
34. What is a DNS server?
A server that stores DNS records and responds to DNS queries,
translating domain names to IP addresses.
35. What is DNS caching?
Temporarily storing DNS query results to speed up subsequent
lookups for the same domain.
36. What is a top-level domain (TLD)?
The last part of a domain name (e.g., .com, .org, .net, .ph).
37. What is a subdomain?
A domain that is part of a larger domain (e.g., mail.example.com,
where "mail" is the subdomain).
38. What is a DNS record?
An entry in a DNS database mapping domain names to IP addresses
or other information.
39. What is an A record?
A DNS record mapping a domain name to an IPv4 address.
40. What is a CNAME record?
Canonical Name record - maps an alias domain name to another
domain name.

41. What is HTTP?

42. What is HTTPS?
HTTP Secure - encrypted version of HTTP using SSL/TLS for secure

43. What is FTP?

44. What is SMTP?
Simple Mail Transfer Protocol - used for sending email messages

45. What is POP3?

46. What is IMAP?
Internet Message Access Protocol - retrieves email while keeping it on

47. What is SSH?

48. What is Telnet?

Network Topologies (51-60)
51. What is a network topology?
The physical or logical arrangement of devices and connections in a
network.
52. What is a star topology?
All devices connect to a central hub or switch, with all tra c passing
through the central device.
53. What is a bus topology?
All devices connect to a single central cable (backbone), with data
transmitted in both directions.
54. What is a ring topology?
Devices connected in a circular chain, with data traveling in one
direction around the ring.
55. What is a mesh topology?
Every device connects directly to every other device, providing
redundancy and reliability.
56. What is a hybrid topology?
A combination of two or more di erent topologies (e.g., star-bus).
57. What is a tree topology?
A hierarchical topology combining multiple star topologies connected
to a central bus.
58. What are the advantages of star topology?
Easy to add/remove devices, failure of one device doesn't a ect
others, centralized management, easy troubleshooting.
59. What are the disadvantages of star topology?
Central device failure brings down entire network, requires more
cable than bus topology, more expensive.
60. What is the di erence between physical and logical topology?
Physical topology describes physical layout of cables and devices.
Logical topology describes how data ows through the network
regardless of physical arrangement.

Network Models (81-90)
81. What is the OSI model?
Open Systems Interconnection model - a conceptual framework with
7 layers describing how network protocols interact.
82. What are the 7 layers of the OSI model?
1. Physical, 2. Data Link, 3. Network, 4. Transport, 5. Session, 6.
Presentation, 7. Application.
83. What is the TCP/IP model?
A simpli ed 4-layer network model: Network Access, Internet,
Transport, Application.
84. What is the Physical layer?
OSI Layer 1 - handles physical transmission of bits over network
media (cables, wireless signals).
85. What is the Data Link layer?
OSI Layer 2 - provides node-to-node data transfer, error detection,
and MAC addressing.
86. What is the Network layer?
OSI Layer 3 - handles logical addressing (IP) and routing of packets
between networks.
87. What is the Transport layer?
OSI Layer 4 - provides end-to-end communication, reliability (TCP),
and ow control.
88. What is the Application layer?
OSI Layer 7 - provides network services directly to user applications
(HTTP, FTP, SMTP).
89. What layer does a router operate at?
Network layer (Layer 3) - uses IP addresses for routing decisions.
90. What layer does a switch operate at?
Data Link layer (Layer 2) - uses MAC addresses for forwarding
decisions.

100. What is a proxy server?

MEDIUM LEVEL (Questions 101-200)
Advanced IP Concepts (101-110)
101. Explain CIDR notation.
Classless Inter-Domain Routing notation (e.g., 192.168.1.0/24)
speci es an IP address and subnet mask compactly. The /24 indicates
the rst 24 bits are the network portion.
102. What is subnetting?
Dividing a larger network into smaller subnetworks to improve
management, security, and e cient IP address allocation.
103. What is a default gateway?
The router IP address used by devices to send tra c destined for
networks outside their local subnet.
104. What is broadcast address?
An address sending data to all devices on a network segment (e.g.,
192.168.1.255 for 192.168.1.0/24).
105. What is multicast?
Sending data to a speci c group of devices simultaneously, more
e cient than broadcasting.
106. What is unicast?
Sending data from one sender to one speci c receiver, the most
common transmission type.
107. What is anycast?
Routing data to the nearest or best destination among a group
sharing the same IP address.
108. What is IPv4 address exhaustion?
The depletion of available IPv4 addresses, driving adoption of IPv6
and NAT.
109. What is a link-local address?
An IP address valid only on the local network segment, not routed
(169.254.x.x in IPv4, fe80::/10 in IPv6).
110. What is APIPA?
Automatic Private IP Addressing - assigns 169.254.x.x address when
DHCP is unavailable.
Routing Concepts (111-120)
111. What is static routing?
Manually con gured routing table entries specifying paths to
destination networks.
112. What is dynamic routing?
Routers automatically learn and update routes using routing
protocols.
113. What is a routing table?
A data structure storing information about network paths, used by
routers to make forwarding decisions.
114. What is a routing protocol?
A protocol enabling routers to exchange routing information and
learn network topology.
115. What is RIP?
Routing Information Protocol - a distance-vector protocol using hop
count as the metric (maximum 15 hops).
116. What is OSPF?
Open Shortest Path First - a link-state routing protocol using cost as
the metric, e cient for large networks.
117. What is BGP?
Border Gateway Protocol - the exterior gateway protocol routing
tra c between autonomous systems on the Internet.
118. What is the di erence between interior and exterior
gateway protocols?
Interior Gateway Protocols (IGPs) like OSPF and RIP route within an
autonomous system. Exterior Gateway Protocols (EGPs) like BGP
route between autonomous systems.
119. What is route aggregation?
Combining multiple network routes into a single routing table entry
to reduce table size.
120. What is the metric in routing?
A value used by routing protocols to determine the best path (hop
count, bandwidth, delay, cost).
TCP/IP Protocol Suite (121-130)
121. What is the three-way handshake?
TCP connection establishment: (1) SYN from client, (2) SYN-ACK from
server, (3) ACK from client.
122. What is a TCP segment?
The Protocol Data Unit (PDU) at the transport layer containing
headers and data.
123. What is a UDP datagram?
The PDU for UDP, a connectionless message with minimal header
information.
124. What is ow control in TCP?
Mechanism preventing sender from overwhelming receiver using
sliding window protocol.
125. What is congestion control?
TCP mechanism adjusting transmission rate based on network
congestion to prevent packet loss.
126. What is the TCP window size?
The amount of data that can be sent before requiring
acknowledgment, controlling ow.
127. What is packet fragmentation?
Breaking large packets into smaller fragments when the packet size
exceeds the Maximum Transmission Unit (MTU).
128. What is MTU?
Maximum Transmission Unit - the largest packet size that can be
transmitted over a network link (typically 1500 bytes for Ethernet).
129. What is the Time to Live (TTL) eld?
An IP header eld limiting packet lifetime by decrementing at each
router hop, preventing routing loops.
130. What is Quality of Service (QoS)?
Techniques prioritizing certain types of network tra c to ensure
performance for critical applications.

135. What is an MX record?

136. What is a PTR record?

137. What is a TXT record?

138. What is DNSSEC?
DNS Security Extensions - adds cryptographic signatures to DNS
records to prevent spoo ng and cache poisoning.
139. What is DNS cache poisoning?
An attack inserting fraudulent DNS data into a cache, redirecting
tra c to malicious sites.
140. What is round-robin DNS?
A load balancing technique rotating through multiple IP addresses for
a single domain name.

151. What is a VLAN?
Virtual Local Area Network - logically segmenting a physical network

Improve security, reduce broadcast tra c, simplify network

Adding VLAN identi cation (tag) to Ethernet frames to identify which

154. What is IEEE 802.1Q?
The standard protocol for VLAN tagging on Ethernet networks.
155. What is a trunk port?
A switch port carrying tra c for multiple VLANs, typically connecting

156. What is an access port?
A switch port belonging to a single VLAN, connecting end devices.
157. What is inter-VLAN routing?
Routing tra c between di erent VLANs using a router or Layer 3

158. What is a native VLAN?
The VLAN assigned to untagged tra c on a trunk port (default VLAN

159. What is VLAN hopping?
A security attack exploiting VLAN con gurations to gain
unauthorized access to other VLANs.
160. What is voice VLAN?
A separate VLAN dedicated to VoIP tra c for quality and security.

DIFFICULT LEVEL (Questions 201-300)
Advanced Routing (201-210)
201. Explain OSPF areas and their purpose.
OSPF divides networks into areas to reduce routing overhead. Area 0
(backbone) connects all other areas. Hierarchical design improves
scalability by limiting LSA ooding to within areas.
202. What is a stub area in OSPF?
An area that doesn't receive external routes, reducing routing table
size. Uses default route for external destinations.
203. What is route redistribution?
Sharing routing information between di erent routing protocols (e.g.,
OSPF to BGP).
204. Explain BGP path selection criteria.
BGP uses multiple attributes in order: Weight, Local Preference, AS
Path length, Origin, MED, eBGP over iBGP, lowest IGP cost, oldest
route, lowest Router ID.
205. What is BGP route dampening?
Suppressing unstable routes that frequently change to prevent
routing table instability.
206. What is policy-based routing?
Routing tra c based on criteria beyond destination address (source,
application, etc.) for tra c engineering.
207. What is MPLS?
Multiprotocol Label Switching - uses labels rather than IP addresses
for forwarding, enabling tra c engineering and VPNs.
208. What is an MPLS label?
A short, xed-length identi er prepended to packets for fast
forwarding decisions.
209. What is a Virtual Routing and Forwarding (VRF) instance?
Separate routing table instances on a single router, enabling network
virtualization and isolation.
210. What is ECMP?
Equal-Cost Multi-Path routing - load balancing tra c across multiple
paths with equal cost metrics.
Advanced TCP/IP (211-220)
211. Explain TCP congestion control algorithms.
Algorithms like Slow Start, Congestion Avoidance, Fast Retransmit,
and Fast Recovery dynamically adjust transmission rate based on
network conditions detected through packet loss and ACKs.
212. What is TCP selective acknowledgment (SACK)?
Extension allowing receivers to acknowledge non-contiguous data
blocks, improving e ciency when multiple packets are lost.
213. What is TCP window scaling?
Option allowing window sizes larger than 65,535 bytes for high
bandwidth, high-latency networks.
214. What is Path MTU Discovery?
Process determining the largest packet size that can traverse a path
without fragmentation.
215. What is ECN?
Explicit Congestion Noti cation - allows network devices to signal
congestion without dropping packets.
216. What is TCP Fast Open?
Optimization allowing data transmission during TCP handshake,
reducing latency for short connections.
217. What is TCP keepalive?
Mechanism periodically checking if a connection is still active during
idle periods.
218. What is the SYN ood attack and its mitigation?
DDoS attack exploiting three-way handshake by sending numerous
SYN packets. Mitigations include SYN cookies, rate limiting, and
rewalls.
219. What is TCP reset attack?
Injecting forged RST packets to terminate legitimate TCP connections.
220. What is QUIC protocol?
Quick UDP Internet Connections - transport protocol built on UDP
combining features of TCP and TLS with reduced latency (used in
HTTP/3).
IPv6 Advanced Concepts (221-230)
221. Explain IPv6 address types.
Unicast (single interface), Multicast (group), Anycast (nearest of
group). No broadcast in IPv6.
222. What is IPv6 Stateless Address Autocon guration (SLAAC)?
Hosts automatically con gure IPv6 addresses using router
advertisements without DHCP.
223. What is the IPv6 neighbor discovery protocol?
Replaces ARP, handles address resolution, router discovery, and
duplicate address detection.
224. What is IPv6 header simpli cation?
Fixed 40-byte header with fewer elds than IPv4, improving
processing e ciency. Extension headers handle additional options.
225. What is dual-stack implementation?
Running both IPv4 and IPv6 simultaneously during transition period.
226. What is IPv6 tunneling?
Encapsulating IPv6 packets within IPv4 for transmission across IPv4
only networks.
227. What is 6to4 tunneling?
Automatic tunneling mechanism using 2002::/16 pre x to embed IPv4
addresses.
228. What is IPv6 unique local address (ULA)?
Private IPv6 addresses (fc00::/7) not routable on public Internet,
analogous to RFC 1918 addresses.
229. What is IPv6 temporary addresses?
Privacy extension generating temporary addresses for outbound
connections to prevent tracking.
230. What is IPv6 ow label?
Header eld identifying packet ows for QoS and load balancing
without deep packet inspection.
Network Security Advanced (231-240)
231. Explain the di erence between symmetric and asymmetric
encryption.
Symmetric uses same key for encryption/decryption (fast, e.g., AES).
Asymmetric uses public/private key pairs (slower, e.g., RSA), enabling
secure key exchange and digital signatures.
232. What is Perfect Forward Secrecy (PFS)?
Generating unique session keys that can't be compromised even if
long-term keys are later exposed.
233. What is a man-in-the-middle (MITM) attack?
Intercepting and potentially altering communication between two
parties without their knowledge.
234. What is certi cate pinning?
Associating a service with speci c certi cate or public key to prevent
MITM attacks using fraudulent certi cates.
235. What is a rainbow table attack?
Using precomputed hash tables to crack password hashes. Defeated
by salting.
236. What is salt in password hashing?
Random data added to passwords before hashing to prevent rainbow
table attacks and ensure identical passwords have di erent hashes.
237. Explain defense in depth strategy.
Layered security approach using multiple overlapping defensive
mechanisms, so single point failure doesn't compromise security.
238. What is network access control (NAC)?
Verifying device compliance with security policies before granting
network access.
239. What is 802.1X authentication?
Port-based network access control using EAP for authenticating
devices before allowing network access.
240. What is RADIUS?
Remote Authentication Dial-In User Service - centralized
authentication, authorization, and accounting (AAA) protocol.
SDN and Network Virtualization (241-250)
241. What is Software-De ned Networking (SDN)?
Separating network control plane (decision-making) from data plane
(packet forwarding), enabling centralized programmable control.
242. What is OpenFlow?
Protocol enabling SDN controllers to communicate with and program
network switches.
243. What is a network controller in SDN?
Centralized software managing network behavior, making routing
decisions and pushing ow rules to switches.
244. What is network function virtualization (NFV)?
Virtualizing network services ( rewalls, load balancers) as software

245. What is a virtual switch?
Software-based switch in virtualized environments connecting
virtual machines.
246. What is VXLAN?
Virtual Extensible LAN - encapsulation protocol creating Layer 2
overlay networks over Layer 3 infrastructure, supporting 16 million
virtual networks.
247. What is network overlay?
Virtual network built on top of physical network infrastructure,
abstracting underlying topology.
248. What is SD-WAN?
Software-De ned Wide Area Network - applying SDN principles to
WANs for intelligent tra c routing and centralized management.
249. What is intent-based networking?
Network management approach where administrators specify
desired outcomes; system automatically con gures devices to achieve
them.
250. What is network automation?
Using scripts and tools to automatically con gure, manage, and
optimize networks, reducing manual intervention.
QoS and Tra c Management (251-260)
251. Explain Di Serv (Di erentiated Services).
QoS architecture marking packets with DSCP values to indicate
priority, with routers providing di erent service levels per class.
252. What is IntServ (Integrated Services)?
QoS model using RSVP to reserve resources for ows, providing
guaranteed service but limited scalability.
253. What is tra c shaping?
Controlling network tra c rate to meet desired bandwidth pro le,
smoothing bursts.
254. What is tra c policing?
Enforcing tra c rate limits by dropping or marking excess packets
exceeding policy.
255. What is the leaky bucket algorithm?
Tra c shaping algorithm releasing packets at constant rate
regardless of input rate, smoothing bursts.
256. What is the token bucket algorithm?
Tra c shaping allowing bursts up to bucket capacity while
maintaining average rate.
257. What is weighted fair queuing (WFQ)?
Scheduling algorithm allocating bandwidth proportionally to ow
weights, preventing starvation.
258. What is priority queuing?
Always serving higher-priority queues rst, potentially starving low
priority tra c.
259. What is class-based queuing (CBQ)?
Hierarchical queuing assigning bandwidth guarantees to tra c
classes with borrowing between classes.
260. What is RED (Random Early Detection)?
Congestion avoidance algorithm randomly dropping packets before
queue lls, signaling congestion to TCP.
Advanced Wireless (261-270)
261. What is Wi-Fi 6E?
Extension of Wi-Fi 6 (802.11ax) into the 6 GHz band, providing more
channels and less interference.
262. What is OFDMA?
Orthogonal Frequency Division Multiple Access - dividing channels

in Wi-Fi 6.
263. What is BSS coloring?
Wi-Fi 6 feature marking frames with color codes to distinguish
between overlapping networks, reducing interference.
264. What is target wake time (TWT)?
Wi-Fi 6 feature scheduling when devices wake to transmit/receive,
improving battery life.
265. What is spatial reuse?
Allowing simultaneous transmissions on same channel in di erent
areas, increasing network capacity.
266. What is the CSMA/CA protocol?
Carrier Sense Multiple Access with Collision Avoidance - wireless
medium access control using RTS/CTS to avoid collisions.
267. What is the exposed node problem?
A node unnecessarily deferring transmission because it hears an
unrelated transmission.
268. What is wireless mesh networking?
Topology where access points connect to each other wirelessly,
extending coverage without wired backhaul.
269. What is the captive portal?
Web page displayed before network access is granted, used for
authentication or terms acceptance.
270. What is Wi-Fi Direct?
Technology allowing direct device-to-device connections without an
access point.
Network Protocols Advanced (271-280)
271. What is RSVP?
Resource Reservation Protocol - signaling protocol reserving network

272. What is SCTP?
Stream Control Transmission Protocol - transport protocol combining

streaming.
273. What is SIP?
Session Initiation Protocol - signaling protocol for VoIP, establishing,

274. What is RTP?
Real-time Transport Protocol - delivering audio and video over IP

275. What is RTCP?
RTP Control Protocol - monitors RTP sessions, providing feedback on

276. What is H.323?
Suite of protocols for multimedia communication over packet

277. What is LDAP?
Lightweight Directory Access Protocol - accessing and maintaining
distributed directory services (like Active Directory).
278. What is Kerberos?
Network authentication protocol using tickets and symmetric
cryptography to verify identities.
279. What is OAuth?
Authorization framework allowing third-party applications limited
access to user resources without sharing credentials.
280. What is SAML?
Security Assertion Markup Language - XML-based standard for
exchanging authentication and authorization data for single sign-on.

289. What is intent-driven networking?

Using quantum mechanics principles for secure communication and
